# 파워리프팅 루틴 생성기 — 진행 상황 & 로드맵

> 최종 갱신: 2026-06-28 · 라이브: https://aa26178787.github.io/powerlifting/ · 저장소: github.com/Aa26178787/powerlifting

근거 기반 개인화 파워리프팅 루틴 생성기. 규칙 기반(결정론적·오프라인·무료) 엔진 + React 웹앱. 모든 수치는 메타분석/RCT/코칭 컨센서스에 근거하며 한계를 UI에 정직하게 고지.

## 1. 빌드 완료 (라이브)

### Phase 1 — 기반 엔진 + 웹앱

### v3 — 설문 위저드 + 진단 + 자질 모델
- **SP1 — 4-자질 모델**: Power/Strength/Hypertrophy/Endurance 블렌드(슬라이더+프리셋), 자질→zone 매핑, 주기화 모델, 오토레귤레이션 우선(무게=제안치), MRV 캡

- **보조운동 세트 구조**: 반복+RPE(체감) 구조 배정
- **템포 표기**: `템포 3-1-1초 (하강-정지-상승)` + T2K(Tempo to Knees) 데드
- **운동 단위 제외**(다중 checkbox) + 리프트별 변형 오버라이드
- **느낌(큐) 약점 → 교정 변형**: 13종 큐(지면추진/힙슈팅/무게중심/등 사용/레그드라이브→T2K 등)
- **SP3 — 일일 체크인 readiness**: 수면/스트레스/피로/부위 상태 → 그날 부하·볼륨 자동 조정 + 통증 부위 드롭/교체, 과피로 트렌드 모니터 배너
- **세트별 RPE 누적 피로**: 같은 무게 세트는 RPE 점증(마지막 세트=목표)
- **kg ↔ lbs 단위 변환**: 입력·표시·CSV 전역

### v4 — 모델 현실화·차별화·실전 디테일 (2026-06-27~28, 다관점 토론 기반)

- **실전 디테일 4종**: 워밍업 세트 자동생성, 세트 간 휴식시간, 운동명 한글화(205/205), 로깅→다음 사이클 피드백(EWMA+클램프, generate 무수정)
- 전부 다관점 토론(3관점→합성) → 배치 구현(TDD·비트동일 가드) → 정직고지 → `docs/research/` 결정 기록. 미지정/기본값이면 기존 동작 비트동일

## 3. 알려진 한계 / 정직 고지 (의도된 범위)
- 변형→메인 전이 수치 미측정(코칭 컨센서스), 강도 수준=대회 상대값(임상 등급 아님)
- 특수 세트 구조는 스트레이트 대비 효과 차이 작음(g=0.159) — 다양성·시간효율·피로관리 목적
- 큐·오버라이드·변형은 **변형 슬롯에만** 적용 → 데드는 3일/주에선 컴페티션 전용이라 4일+부터 반영
- readiness 환산은 휴리스틱(개인 보정 필요), 과피로 경보=단순 트렌드(진단 아님, ACWR 미사용)
- 부위 통증 교체(status 2)는 단일 pick을 부위로만 필터(스타일 무시)
- 다종목 묶음(콘트라스트/슈퍼/자이언트)은 표시·메모만(완전 오케스트레이션 후속)
- VBT(영상 속도) 모듈은 보류(근거 약함 — 속도손실 밴드 + 개인 보정으로 재설계 시 재검토)
- 동시(근력 탑+근비대 백오프) 구조와 나이 볼륨 감소는 코칭 컨센서스(RCT 아님)
- 성별·체중은 상대강도 진단 전용 — 프로그래밍 부하 미반영(근거 약함, 의도된 한계)
- 보조운동 머신/프리웨이트 선호 = 효과 유사, 선호·안정성 선택(토글 제공)
- 변형 우선순위 = 표준 변형 우선(코칭 컨센서스), 특수 변형은 목적형
- 종목별 빈도 분배 = 균등 간격 휴리스틱(개인 회복차 있음), 종목당 컴페티션 1일 + 변형
- 메인 종목 세션당 절대 세트 상한(스쿼트6·벤치8·데드4) — 데드 최저는 축성/중추 피로 합의, 정확 수치는 휴리스틱. 주간 볼륨이 상한 초과 시 한 세션에 몰지 않음(빈도 추가 권장)
- 고반복 RPE→%1RM 보정(+2~4%p) — 환산표는 저반복만 정확, 고반복 과소평가를 보정해 라벨된 RPE와 실부하 일치(보정폭은 추정치)
- 주차별 탑세트 부하 점진 진행(최대 +4%, 목표 RPE 일정, 디로드 리셋) — 점진 과부하 원칙(ACSM 2~10%) 기반, 주간 증가폭은 교과서 관례(근거 약함)
- 데드리프트 주간 볼륨을 다른 종목의 ~60%로 낮춤 (축성 피로·느린 회복 반영 — 코칭 컨센서스, 정확 계수 미검증)
- 보조운동 개수는 세션 시간에서 메인 소요를 빼고 목표(근비대↑/근력↓)에 따라 조정 (휴리스틱)
- 피킹(피크) 단계에서 탑 싱글이 시합 강도(~95-98%1RM)로 점진 상승 (강도 특이성 SAID 기반; 정확 % 컨센서스)
- 종목별 주간 세트는 실제 전달량으로 표기(저빈도 종목은 세션당 상한 때문에 목표보다 적게 전달될 수 있음)
- 볼륨 테이퍼(대회 주기)·완만 누적(근력 위주)의 방향성은 코칭 컨센서스/교과서 근거이나, 피크주 −45%·전환 시점(2/3)·완만 계수(+0.20)는 검증된 수치가 아닌 휴리스틱(`근거 약함`). 비대회 사용자에게 테이퍼 미적용
- 근육별 볼륨 요약은 per-lift 처방에서 파생된 보고이지 측정된 생리 데이터가 아님. per-muscle 랜드마크·협력근 0.5 분할신용·15군 분류는 per-lift 밴드보다 덜 검증된 휴리스틱(`근거 약함`). 파워리프팅 사용자에게는 SBD 특이성 보호를 위해 미달 근육 채우기(deficit-fill) 미적용
- 보조운동 근육 타깃팅 강도는 휴리스틱(consensus 외삽): 파워빌딩은 미달 근육군 deficit-fill 활성(PL은 SBD 특이성 보호로 0). 파워빌딩은 또한 보조 **개수**도 +1 슬롯(goalBias −1→0, hyp 점유율 활성 영역) — 그라데이션 PL(−1) < PB(0) < 보디빌딩(+1); 파워·athletic은 hyp 낮아 −1 유지. 방향 근거(부하 무관 근비대, 수확체감 — Pelland/Zourdos 2026; Schoenfeld 2017); 정확 계수(full 0.6 / 활성 시작 hyp점유율 0.30 / 포화 0.50)·개수 바이어스·MEV/MAV/MRV 밴드는 컨센서스·추정치이며 직접 임상 임계 아님
- 피킹 플랜에서 보조운동 deficit-fill 강도를 phase별로 단계 감소(축적 1.0→강화 0.5→피크 0.0)하고 피크 주 보조 개수를 1개 절감. 방향은 블록 주기화 컨센서스(Issurin 2010; Mujika &amp; Padilla 2003)이나 정확 승수·감축폭·phase 임계는 휴리스틱(`근거 약함`). floor=1로 피크 주에도 보조가 0이 되지 않음(순수 PL 피커도 peak주 보조 2→1 taper); 비대회 플랜 bit-for-bit 보존
- 메인 강도 차별화: 근력 비중(`strengthShare = str/(str+hyp)`)으로 strengthHypertrophy 스킴의 heavy:moderate 세트 분할 + concurrent 출현 빈도 결정. PB는 중강도 백오프 비중↑로 평균 강도↓(실측 heavy비율 PL 79% vs PB 66%), top-end ~92%는 둘 다 보존. 차별화는 concurrent 경로(혼합 blend)에만 작용 → 순수 PL/power-dom bit-identical. 방향=메타/컨센서스(근력 부하의존·근비대 부하무관), 정확 비율·HEAVY_FLOOR 0.40·CONC_DENOM 3=휴리스틱(`근거 약함`)
- 실패 근접도(proximity-to-failure): 근비대 세트를 근력 세트보다 실패에 더 가깝게 처방(목표 RPE 9 vs 8.5). 방향은 2024 메타분석 근거(`근거 중`), 정확한 RPE 값은 추정치(`근거 약함`)
- 훈련 빈도 차별화: 빈도는 근력 향상에 독립적으로 기여(메타분석, `근거 강`)하지만 근비대에는 같은 볼륨이면 영향 작음(볼륨 분배 수단, `근거 중`). 근력 위주 블렌드에 빈도 +1 세션 권장(바이어스 수치는 추정치, `근거 약함`)
- 주기화 모델 동등성: 선형/비선형/블록 선택은 볼륨이 같으면 결과 차이 거의 없음 — 볼륨·강도·실패 근접도가 더 중요(`근거 중`). 모델 간 우열 근거 약함(`근거 약함`)
- 약점/우선 종목 선행(lagging-first): 약점·우선 종목 보조운동을 세션 앞에 배치. 첫 운동이 적응 이득 큼 — 코칭 컨센서스(`근거 중`)
- 긴 근육 길이(스트레치) 강조: 근비대 보조에 스트레치 포지션 강조 안내. 방향은 최근 근거(`근거 중`), 종목 선정은 휴리스틱(`근거 약함`)
- 16~24주 블록 주기화: ~6주마다 자동 디로드 + 블록별 부하 램프 리셋. 방향은 블록 주기화 컨센서스(`근거 중`), 정확한 6주 주기는 휴리스틱(`근거 약함`).
- 대회 피킹 플랜 최종 주 realization 테이퍼(강도 유지, 볼륨 ~40% 감소). 방향은 Bosquet 메타분석(`근거 강`), 정확한 감소 계수는 휴리스틱(`근거 약함`). 비대회 플랜 디로드는 회복형(RPE 6) 유지.
- e1RM ±20% 일일 변동 밴드 표시(처방 부하는 고정). 권고적 advisory — 사용자가 당일 수행 범위 인식용.
- 단조로움(monotony)·strain 지표로 과훈련 경향 모니터링(Foster — 방향은 `근거 중`, RPE×반복 부하 프록시는 `근거 약함`). 추세 표시로 readiness와 상호 확인용.
- 예상 피크 시점은 Fitness-Fatigue 모델(Banister) 기반 추정(모델 방향 `근거 중`, 개인 파라미터 `근거 약함`). 보조 지표로 사용자 감각과 병행.
- ACWR(급성:만성 부하비) 표시만 제공하며 기준으로 미사용(`근거 약함` — 신뢰성 논란). 참고용 illustrative only.
- **오버로딩 모드**는 의도적 과부하 도박수이며, 결과는 보장되지 않습니다. 목표 상승폭(targetPct)은 공격성 조절 입력일 뿐이며, 실패 시 정체·부상·번아웃 위험이 있습니다. 선택 종목은 의도적으로 MRV를 초과하며, 마지막 주에 realization 테이퍼(강도 유지, 볼륨 감소) 후 테스트합니다(`근거 중` — 과부하 원칙은 명확하나 개인 회복·부상 위험 추정은 변수 큼).
- 오버로딩 dose·위험도·쿨다운·기대값 계수는 코칭 컨센서스 방향을 따르지만, 정확 계수는 미검증 휴리스틱입니다(`근거 약함`). 프리셋 중 Smolov Jr·Russian Squat 외에는 원 프로그램의 정확한 세트/렙 표가 아닌 파라미터 시드이며, ACWR은 오버로딩 판정 게이트로 쓰지 않습니다.
- 데드리프트 반복 상한: 모든 데드리프트 세트를 6회 이하로 제한하고 부하를 그에 맞춰 재계산(같은 RPE에서 저반복 = 고중량). 고반복 데드의 축성/CNS 피로·자세 붕괴를 피하기 위함. 방향은 코칭 컨센서스(`근거 중`), 정확한 상한(6회)은 휴리스틱(`근거 약함`). 다른 종목 미적용·비트동일.
